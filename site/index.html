Local workflow

From dropped file to clean text, on your machine.

A product-direction preview for the local workflow AutoWhisper is being shaped around. Release notes remain the source of truth for what ships in the current macOS build.

Privacy posture

A clear boundary: your speech does not need a remote transcription service.

AutoWhisper is positioned around local processing, local models, and a desktop workflow that does not require cloud speech APIs for normal dictation. The transcription core runs on whisper.cpp on your device.

The claim is intentionally narrow: local-first transcription, not magic security dust. You still control where the resulting text is pasted, saved, or sent. Every release links to a notarized artifact you can verify before installing.

  • Model runtimewhisper.cpp on device
  • Network during dictationnone required
  • macOS distributionDeveloper ID signed and notarized
  • Linux distributionPPA package, source-built

Why locally

Hours of audio. Minutes to text.

Keep the model close to your microphone, your files, and your network. The current release proves the local dictation core; the broader file, library, and queue workflow is the design direction for the app family.

  • 0 cloud calls during normal dictation
  • Queue direction designed for restart-safe batches
  • Index direction designed to stay beside your files

Linux install

Ubuntu / Debian via PPA

Add the package source, refresh apt, then install the desktop utility.

sudo add-apt-repository ppa:primemanifold/autowhisper
sudo apt update
sudo apt install autowhisper

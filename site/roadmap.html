Public roadmap

What is shipping, what is queued.

AutoWhisper is a small desktop tool kept on a deliberate cadence and held to a local-first promise. This page lists the work that is in motion, the work waiting in line, and the work that has already shipped.

Now

In flight

  • macOS onboarding refinement

    Tighter first-run flow for the notarized v0.7.0 build, including microphone permission prompts and clearer hotkey discovery.

  • Whisper model picker

    Choose a local Whisper model size from the menu bar without touching configuration files by hand.

Next

Queued

  • Linux desktop polish

    Hotkey reliability under Wayland and a packaged AppIndicator for the PPA install path.

  • Custom dictionary

    Pin proper nouns, command names, and acronyms so transcription respects local vocabulary.

Later

On the horizon

  • Per-app profiles

    Different prompt and formatting defaults when dictating into a terminal, an editor, or a chat client.

  • Voice command actions

    Trigger small local actions, such as inserting templates, while keeping audio on the device.

Recently shipped

Landed in the last few releases.

  • v0.7.0 notarized macOS build

    Developer ID signed, notarized, and stapled, with Gatekeeper acceptance verified on a clean machine before the release was tagged.

  • Ubuntu and Debian PPA

    Install via sudo apt install autowhisper after adding the PPA, with no manual binary handling required.

  • Local-first transcription core

    Push-to-talk dictation routed entirely through whisper.cpp on device, keeping the local-first contract intact with no remote transcription dependency.
